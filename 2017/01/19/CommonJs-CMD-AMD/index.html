CommonJs-CMD-AMD

Posted on 2017-01-19   |   In front-end   |     |   visitors

区分三者

关于Js中的模块化规范(CommonJS，AMD，CMD),很多人可能分不清三者之间到底是什么关系,下面我来梳理梳理,let’s get started.

CommonJs

由于Js语言本身设计时间仓促,缺少类等(ES5),没有模块化的思想,在一个js中不能引入另一个js文件.开发人员就为js的模块化制定了一套规范,取名CommonJs.而NodeJs是这种规范在后台的实现.

注意,CommonJs加载模块是同步的,只有加载完成一个模块后才能执行后面的操作.而Node.js主要用于服务器的编程,加载的模块文件一般存在本地,加载快,不用考虑异步加载的方式.所以CommonJs规范比较适用.but呢,如果实在浏览器环境,我们要从服务器中加载模块,这里就必须采用异步模式. 于是,在前端产生了AMD和CMD两种方案.

AMD(Asynchromous Module Definition)

  • AMD 是 RequireJS 在推广过程中对模块定义的规范化产出
  • AMD异步加载模块。
    它的模块支持对象 函数 构造器 字符串 JSON等各种类型的模块。
